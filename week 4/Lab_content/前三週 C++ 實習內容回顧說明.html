前三週 C++ 實習內容回顧說明

前面三個星期的實習內容主要目的是讓大家趕快進入狀況, 主要內容包括下列:

  1. 打字, 多檔案編譯, #include 說明, 記憶體檢測工具

  2. iostream 練習, string 類別練習, vector 類別練習

  3. 函式指標
    • stdlib sort() intCompare()
    • std::find() operator==()
    • SaleItem::print()
其中前兩者是希望大家趕快回憶起 C 和 Data Structure 課程學到的內容, 建立以 C++ 學習物件導向程式製作的基礎。

也許你覺得這些練習好像都沒有完整地在課堂裡講過, 只有提到比較基本的語法和概念, 就要你來實習課上機寫了, 好像不是那麼容易! 的確是這樣的, 不過換一個角度想, 如果都只練習你已經比較熟悉的東西, 那實習課應該是蠻浪費時間的, 好像只是工匠在熟悉技藝一樣, 再提醒大家一下, 大學裡你應該學到的不是只有各種技術, 你應該要學習怎樣找到關鍵的技術, 怎樣學習關鍵的技術。

我對實習課的要求是這樣的:

我希望在實習課裡提供你足夠份量發揮的內容, 你以自己的進度來學習 (這和課堂上課有很大的差別), 在你動手的同時, 你可以不斷地透過實作驗證自己的認知, 也提供你助教, 同學, 老師來詢問, 修正你自己吸收技術的方法, 讓這個程式設計的課程不會淪於空洞, 當然你很快也就會發現個人學習的動力其實主宰你在實習課學習的內容, 這也是資訊系學生將來成功地面對工作挑戰時很關鍵的心理素質。
"函式指標的練習" 希望你藉由給你的三個範例體會到一個以前可能沒有練習過的概念:
"我們除了可以透過函式的參數把一些資料傳遞給函式處理之外, 也可以把一些處理的步驟透過函式的參數傳遞給函式"
同樣的一個函式, 除了可以處理傳入的各種不同資料之外, 也可以運用傳入的不同的處理方法來處理資料(而且不需要事先就寫好在函式裡), 甚至在第三個練習裡, 你可以發現在傳遞資料給一個函式的同時, 處理這個資料的方法也和資料綁在一起傳遞給函式, 這是 C 語言在支援物件化程式和程序化的程式時一個很大的區別, 你以後還會看到 C++ 中用更方便的方法來實作這種概念, 所以在這裡讓大家重複地練習了三個子單元。
